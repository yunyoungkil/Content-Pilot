## **Content Pilot: AI 진화 전략 기획서 (v2.0)**

### **1. 🎯 최종 목표 (Ultimate Goal)**

Content Pilot의 AI를 단순한 '분석 도구'에서, 사용자와 함께 성장하며 \*\*콘텐츠 비즈니스의 수익화를 돕는 '전략적 파트너'\*\*로 진화시키는 것을 목표로 한다. '감'이 아닌 데이터 기반의 의사결정을 통해 콘텐츠의 성공 확률을 극대화하는 완결된 생태계를 구축한다.

### **2. 🔁 핵심 개념: AI 자가 학습 및 진화 사이클**

AI의 제안을 \*\*'가설'\*\*로 보고, 사용자의 발행 및 성과 데이터를 \*\*'실험 결과'\*\*로 활용하는 선순환 구조를 의미한다. 이 사이클은 4단계로 구성된다.

1.  **가설 설정 (AI 제안)**: AI가 현재까지 수집된 데이터를 기반으로 성공 확률이 높은 콘텐츠 아이디어를 제안한다.
2.  **실험 (사용자 실행)**: 사용자가 AI의 제안을 채택하여 실제 콘텐츠를 발행한다.
3.  **결과 분석 (성과 측정)**: 시스템이 발행된 콘텐츠의 실제 성과(조회수, 체류 시간, 수익 등)를 자동으로 측정하고, '아이디어-성과' 데이터를 연결한다.
4.  **재학습 및 고도화**: AI가 '성공/실패'로 검증된 실험 결과를 학습하여, 다음 아이디어 제안(가설 설정) 시 더욱 정교하고 성공 확률 높은 제안을 하도록 스스로를 진화시킨다.

### **3. 📋 기능 요구 사항 (PRD)**

기존 기능(G-1 \~ G-12)을 확장하고 AI 진화 사이클을 구현하기 위한 신규 기능 목록입니다.

| ID | 유형 | 기능명 | 설명 |
| :--- | :--- | :--- | :--- |
| **B-1** | **추가** | **AI 블로그 성과 분석** | '내 채널'의 블로그 게시물 데이터를 AI로 분석하여 성공 요인과 개선점을 리포트 형식으로 제공합니다. |
| **B-2** | **추가** | **AI 블로그 경쟁 비교 분석** | 내 블로그와 경쟁 블로그의 인기 포스트를 비교 분석하여, 두 채널의 강점을 결합한 새로운 블로그 아이디어를 제안합니다. |
| **G-13** | **추가** | **외부 데이터 커넥터 (Google)** | OAuth 2.0 인증을 통해 사용자의 구글 애널리틱스(GA4), 구글 애드센스 계정 데이터를 안전하게 연동하는 기능을 제공합니다. |
| **G-14** | **추가** | **콘텐츠별 심층 지표 수집** | 발행된 콘텐츠 URL별로 페이지뷰, 평균 체류 시간, 유입 경로, 예상 수입, 페이지 RPM 등 \*\*수익과 직결되는 핵심 성과 지표(KPI)\*\*를 API를 통해 주기적으로 수집하고 저장합니다. |
| **G-15** | **고도화** | **수익화 기반 AI 분석 모델** | 기본 성과 데이터와 함께 **G-14**에서 수집된 심층 지표를 AI 프롬프트에 포함시킵니다. AI가 '어떤 콘텐츠가 실제 수익으로 이어지는지'를 학습하고, 트래픽 품질과 수익성을 함께 고려한 전략적 아이디어를 제안하도록 합니다. |
| **G-16** | **추가**| **아이디어-성과 데이터 연결**| '기획 보드'의 카드와 실제 발행된 콘텐츠의 URL을 1:1로 매칭하고, 수집된 성과 데이터를 이 연결고리에 귀속시켜 AI의 학습 데이터로 활용합니다. |
| **G-17**| **추가**| **사용자 피드백 기록**| 사용자가 AI의 제안을 '채택'하거나 '무시'하는 행동을 긍정/부정 신호로 데이터베이스에 기록하여, AI가 사용자의 암묵적인 선호도를 학습하도록 합니다. |

### **4. 🚶‍♂️사용자 행동 패턴 (User Flow)**

AI 진화 사이클이 적용된 최종 사용자 시나리오입니다.

| 단계 | 사용자 행동 | 프로그램 반응 (최적화된 기능) |
| :--- | :--- | :--- |
| **1. 데이터 연동** | '채널 연동' 탭의 \*\*'고급 데이터 연동'\*\*에서 'Google 계정으로 로그인'하여 애널리틱스와 애드센스 데이터 접근 권한을 승인합니다. | OAuth 2.0으로 데이터 소스가 안전하게 연결되고, `background.js`는 해당 계정의 데이터를 조회할 권한을 얻습니다. |
| **2. 분석 및 아이디어 포착** | '대시보드'에서 '성과 분석'을 실행하고, AI가 제안한 아이디어 중 마음에 드는 항목의 **'📌 기획 보드에 추가'** 버튼을 클릭합니다. | 1.  **(아이디어 카드 생성)** 해당 아이디어가 기획 보드 '아이디어' 컬럼에 카드로 추가됩니다. (**피드백 기록: 긍정 신호**)<br>2.  **(자동 아카이빙)** **첫 번째 아이디어를 추가하는 순간**, AI 분석 리포트 전체가 `#AI리포트` 태그와 함께 **스크랩북에 자동으로 저장**됩니다. (이후 중복 저장 안 됨) |
| **3. 콘텐츠 발행 및 성과 추적 연결** | '기획 보드'의 카드를 바탕으로 콘텐츠를 제작/발행한 후, 해당 카드를 '발행 완료' 컬럼으로 옮기고 **발행된 URL을 입력**한 뒤 \*\*'성과 추적 시작'\*\*을 클릭합니다. | **(데이터 연결)** 'AI 추천 아이디어'와 '실제 발행물'을 1:1로 매칭하고, 이때부터 해당 URL의 **기본 성과(조회수)와 고급 성과(수익, 체류 시간 등)를 주기적으로 수집**합니다. |
| **4. AI 재학습 및 진화된 제안 확인** | 며칠 후, 다시 '성과 분석'을 실행합니다. | **(진화된 분석)** AI는 이제 \*\*"과거에 성공/실패했던 아이디어의 실제 성과 데이터"\*\*와 \*\*"사용자의 채택/무시 기록"\*\*까지 모두 고려하여, **수익화에 초점을 맞춘 훨씬 더 정교하고 전략적인 아이디어를 제안**합니다. |
